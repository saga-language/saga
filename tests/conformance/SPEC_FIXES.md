# Proposed Spec Changes

Entries here are conformance-test failures where the working theory is
that **the spec is wrong**, not the implementation.

Nothing in this file gets applied to `docs/language.md` automatically.
Each entry is a proposal awaiting an explicit human decision.

Entry format:

```
## <short title>

- Spec source: `docs/<file>.md:<line>` — quote the offending claim
- Conformance test: `tests/conformance/<section>/<test>.sg`
- Why the spec is suspected wrong: <reason — design moved, contradicts
  another section, was aspirational, etc.>
- Proposed change: <what the spec should say instead>
```
